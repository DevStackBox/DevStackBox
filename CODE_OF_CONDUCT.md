# Code of Conduct

DevStackBox is an open-source project. This document explains the expectations for everyone participating in the community, including issues, pull requests, discussions, and other project spaces.

## Be decent

- Be respectful. Disagree with ideas, not people.
- Assume good intent. A lot of contributors are learning.
- Keep comments useful. Long rants and personal attacks waste everyone's time.
- No harassment, hate speech, or discrimination of any kind.
- Don't spam, derail threads, or push people into off-platform contact they didn't ask for.

If you wouldn't say it to someone's face at a meetup, don't post it here.

## What this covers

This applies to:

- GitHub issues, PRs, and discussions
- Comments on commits and reviews
- Project docs and community spaces tied to DevStackBox

It also applies when you are representing the project elsewhere (for example, answering questions about DevStackBox).

## Reporting a problem

If someone is being abusive or making the project a bad place to contribute, email **hello@devstackbox.com** with:

- What happened
- Where it happened (link if you have one)
- Any extra context that helps

You can ask for the report to stay private. We will look into it and respond as soon as we can. We won't retaliate against people who report issues in good faith.

## Enforcement

Depending on how serious it is, we may:

1. Ask you to stop and edit or delete the comment
2. Give a warning
3. Temporarily block you from the repo
4. Permanently ban you from participating

For one-off mistakes, a warning is usually enough. Repeated or severe behavior gets a harder response.
